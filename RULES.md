# Rules

Always read this file before writing code.

## General Principles

- Prefer the simplest working implementation.
- Prefer concise code written by experienced engineers.
- Avoid unnecessary abstractions, hooks, layers, or files.
- Avoid overengineering.
- When in doubt, choose the simpler solution.

## Code Structure

- Keep functions small and readable.
- Split files only when logic is reusable or the file becomes large.
- Avoid excessive helper functions.
- Prefer clarity over cleverness.

## Complexity Control

- Minimize state and side effects.
- Avoid deep nesting and overly complex logic.
- Prefer straightforward solutions over generalized ones.

## Dependencies

- Avoid adding new libraries unless they provide clear value.
- Prefer built-in language or platform features.

## Performance Awareness

- Avoid unnecessary computation.
- Prefer predictable and efficient execution paths.

## Maintainability

- Code must be easy to read, modify, and debug.
